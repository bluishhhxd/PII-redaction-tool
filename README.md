# PII Redaction Tool

This project redacts PII from the provided Word document and writes a redacted `.docx` output with fake but consistent alternatives.

## Files

- `redact_pii.py` - source code for the redaction script.
- `Red_Herring_Prospectus_Redacted.docx` - generated redacted output.
- `evaluation_report.md` - evaluation method, run summary, and metrics.
- `requirements.txt` - notes that the script has no external dependency.
- `redaction_summary.json` - non-sensitive run summary generated by the script.

## How to run

```bash
python redact_pii.py "Red Herring Prospectus.docx" -o "Red_Herring_Prospectus_Redacted.docx" --summary-json redaction_summary.json
```

The script uses only Python's standard library, so there are no package installs required.

## Approach

The solution edits the `.docx` package directly by reading WordprocessingML XML from the body, headers, footers, footnotes, endnotes, and comments. It detects PII with regex and context-aware heuristics, then replaces each unique original value with a deterministic fake value so repeated names, companies, emails, phones, and addresses stay consistent throughout the output.

Covered PII types include full names, email addresses, phone numbers, company names, physical/mailing addresses, SSNs, credit cards, dates of birth, and IP addresses. I also redact URLs/domains because the prospectus website reveals the original company identity.

## Tradeoffs

This is a rule-based solution rather than an ML/NER model, which keeps it portable and easy to inspect. The main tradeoff is that unusual person names without context may be missed, while public company names and URLs may be intentionally over-redacted because company names are in scope for the assignment. Regular dates are not redacted unless they appear in a date-of-birth context, which avoids masking prospectus dates, filing dates, and financial period dates. The script does not OCR text embedded inside images.
